# 08 ポストモーテム

対応講義: `s9-l3` ワーク: 模擬インシデントのポストモーテムを書く

## 目的

模擬インシデントで記録したタイムラインを使い、責任追及ではなく改善につながるポストモーテムを書きます。

## 手順

1. [ポストモーテムテンプレート](../postmortem-template.md) を開く
2. タイムラインを埋める
3. ユーザー影響を数字で書く
4. 根本原因を人のミスで終わらせない
5. 再発防止策に担当者と期限を入れる

## 書く内容

- 概要
- 影響範囲
- タイムライン
- 根本原因
- 寄与要因
- よかった点
- 改善点
- アクションアイテム

## 期待結果

「CPUが高かった」で終わらず、次の改善行動まで書けている状態にします。

良いアクション例:

- Run Command実行権限を運用ロールに限定する
- CPU高騰時のRunbookを整備する
- SLO違反時のエスカレーション条件を明文化する
